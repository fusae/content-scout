## Phase 7: 反馈学习层

### 核心功能

**FeedbackLearner 类** (`src/feedback/index.ts`)
- 反馈模式分析：提取接受/拒绝的话题和来源
- 画像更新：根据反馈调整主题权重和兴趣标签
- 黑名单管理：连续 3 次拒绝的话题自动加入黑名单
- 学习触发条件：累积 10 次反馈或每周一次

### 学习策略

1. **正向反馈**：
   - 将接受的话题添加到主题列表
   - 更新兴趣标签（最多保留 10 个）
   - 提高相关来源的权重

2. **负向反馈**：
   - 统计拒绝次数
   - 连续 3 次拒绝 → 加入黑名单
   - 降低相关来源的权重

3. **权重调整**：
   - 保守策略：小步调整，避免过度反应
   - 保留历史数据：支持回滚和分析

### 验收标准

✅ 反馈正确记录到数据库  
✅ 画像权重根据反馈调整  
✅ 推荐质量随时间提升  

## Phase 8: 定时任务和工作流

### 核心功能

**Scheduler 类** (`src/scheduler/index.ts`)
- 定时任务调度：基于 node-cron
- 完整工作流编排：聚合 → 画像 → 过滤 → 生成 → 推送
- 错误处理：全局捕获 + 飞书通知
- 手动触发：支持开发调试

### 定时任务配置

1. **每日任务**（9:00）：
   - 内容聚合（7 个平台）
   - 智能过滤（两阶段）
   - 草稿生成（3 种风格）
   - 飞书推送（3-5 条推荐）

2. **每周学习**（周日 2:00）：
   - 分析最近 7 天反馈
   - 更新账号画像
   - 调整推荐策略

3. **每日清理**（3:00）：
   - 删除 7 天前的过期内容
   - 释放数据库空间

### 工作流特性

- **容错机制**：单个步骤失败不影响整体
- **详细日志**：记录每个步骤的耗时和结果
- **性能监控**：统计完整流程耗时
- **错误通知**：失败时通过飞书告警

### 验收标准

✅ 定时任务准时触发  
✅ 完整工作流能在 20 分钟内完成  
✅ 错误能正确处理和通知

## 使用方法

### 启动应用

```bash
# 开发模式（立即执行一次任务）
npm run dev

# 生产模式（仅启动定时任务）
npm start
```

### 手动触发任务

```bash
# 在代码中调用
scheduler.runManually()
```

### 查看日志

```bash
# 实时查看日志
tail -f logs/app.log

# 查看错误日志
tail -f logs/error.log
```

## 环境变量配置

```bash

CRON_WEEKLY_LEARNING=0 2 * * 0      # 每周日 2:00
CRON_CLEANUP=0 3 * * *              # 每天 3:00

# 学习阈值
FEEDBACK_LEARNING_THRESHOLD=10      # 累积 10 次反馈触发学习
```
